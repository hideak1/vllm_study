# 第11部：コントリビュート & 開発ワークフロー

**目標：** vLLM と SGLang の生産的なコントリビューターになる — 開発環境の構築から PR のマージまで。

オープンソースへの貢献にはコードだけでなく、開発ワークフロー全体の理解が必要です。この部では、環境構築、テスト、デバッグ、プロファイリング、コードレビュー、CI/CD を網羅します。バグ修正、機能追加、ドキュメント改善のいずれであっても、効果的に貢献するための実践的なスキルを習得できます。

## 章一覧

| 11.2 | [テストフレームワーク](../notebooks/part11/chapter_11.2_testing.ipynb) | ユニットテスト、結合テスト、モデルテスト、テストフィクスチャ、pytest パターン |

## 前提条件

- 第3〜6部を修了していること（両方のコードベースの理解）
- Git の習熟（ブランチ、リベース、コンフリクト解決）
- Python テスト経験（pytest）
- CI/CD の概念に精通していること

## 学習成果

第11部を修了すると、以下のことができるようになります：

- vLLM と SGLang の完全な開発環境を構築できる
- ユニット、結合、システムレベルでのテストを作成・実行できる
- Python、CUDA、分散実行にまたがる複雑な問題をデバッグできる
- パフォーマンスをプロファイリングし、リグレッションや最適化の機会を特定できる
- CI を通過し効率的に承認される、構造化された PR を提出できる
- 両プロジェクトの CI/CD パイプラインとリリースプロセスを理解できる
