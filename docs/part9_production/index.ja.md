# Part 9: 本番デプロイと運用

**目標:** vLLMとSGLangを本番環境に自信を持ってデプロイ・運用する — インフラ、モニタリング、コスト最適化をカバー。

開発から本番への移行には新たな課題が伴います：信頼性、スケーラビリティ、可観測性、そしてコストです。このパートでは、デプロイオプションやロードバランシングからモニタリングダッシュボード、コスト最適化戦略まで、LLMサービングを大規模に運用するために必要なすべてをカバーします。

## 前提条件

## 学習成果

Part 9を修了すると、以下ができるようになります：

- Docker、Kubernetes、またはベアメタルでvLLMとSGLangを本番デプロイできる
- LLMサービング向けの包括的なモニタリングとアラートを構築できる
- モデルサービングのパフォーマンスを厳密にベンチマークする手法を設計・実行できる
- SLA要件を維持しながらインフラコストを最適化できる
- 大規模モデル向けに複数ノードにまたがるサービングをスケールできる
- オンライン vs オフライン推論ワークロードに適切な設定を選択できる
